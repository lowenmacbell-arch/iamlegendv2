🔐 Getting Your Session ID

[!IMPORTANT]
The bot uses a Session ID to connect without scanning QR every time. Generate it once and paste in .env.

Step 1 – Open the session generator

https://img.shields.io/badge/GET_SESSION_ID-Click_Here-25D366?style=for-the-badge&logo=whatsapp

Step 2 – Generate

Option A – Pair Code (Recommended)

1. Enter your bot's WhatsApp number (e.g. 255618558502)
2. Click Generate Pair Code
3. An 8‑character code appears (e.g. J38K-4PNS)
4. On your phone: WhatsApp → ⋮ Menu → Linked Devices → Link a Device → Link with phone number
5. Enter the code – session is created
6. Copy the Session ID shown on the page

Option B – QR Code

1. Click the QR Code tab
2. Scan with WhatsApp
3. Copy the Session ID shown after scanning

Step 3 – Add to .env

```env
SESSION_ID=Stanytz378/IAMLEGEND_xxxxxxxxxxxxxxxxxxxxxxxx
```

Alternative – Pairing via terminal

Leave SESSION_ID empty and set:

```env
PAIRING_NUMBER=255618558502
```

The bot will print an 8‑character code on startup. Link it within 60 seconds.

⚙️ Configuration

Copy sample.env to .env:


```env
# ── REQUIRED (choose one) ────────────────────────────────────
SESSION_ID=Stanytz378/IAMLEGEND_your_gist_id_here
# OR
PAIRING_NUMBER=255618558502

# ── REQUIRED ─────────────────────────────────────────────────
OWNER_NUMBER=255618558502        # No + sign

# ── BOT IDENTITY ─────────────────────────────────────────────
BOT_NAME=ᴵ ᴬᴹ ᴸᴱᴳᴱᴺᴰ
BOT_OWNER=STANYTZ
PACKNAME=ᴵ ᴬᴹ ᴸᴱᴳᴱᴺᴰ

# ── BEHAVIOUR ────────────────────────────────────────────────
PREFIXES=.,!,/                   # Comma-separated
COMMAND_MODE=public              # public or private
TIMEZONE=Africa/Nairobi

# ── OPTIONAL API KEYS ────────────────────────────────────────
REMOVEBG_KEY=                    # https://remove.bg/api

# ── PERFORMANCE ──────────────────────────────────────────────
PORT=5000
MAX_STORE_MESSAGES=50

# ── DATABASE (all empty = JSON files) ────────────────────────
MONGO_URL=
POSTGRES_URL=
MYSQL_URL=
DB_URL=                          # SQLite: ./data/baileys.db
```

📦 Installation

Manual Install

```bash
git clone https://github.com/Stanytz378/IAMLEGEND.git
cd IAMLEGEND
npm install
cp sample.env .env
nano .env               # edit your details
npm start
```

One‑Line VPS Installer

```bash
sudo bash <(curl -fsSL https://raw.githubusercontent.com/Stanytz378/IAMLEGEND/main/lib/install.sh)
```

This installs Node.js 20, ffmpeg, libvips, libwebp, PM2, clones the repo, and sets up data files.

After install:

```bash
nano /root/IAMLEGEND/.env
cd /root/IAMLEGEND && pm2 start index.js --name iamlegend
pm2 save && pm2 startup
```

🔌 Plugin System

[!IMPORTANT]
Plugins live in plugins/ and are auto-loaded on startup – no registration needed. Each file must export a default object.

Plugin Template

```javascript
export default {
    command: 'mycommand',
    aliases: ['mc', 'mycmd'],
    category: 'utility',
    description: 'Does something cool',
    usage: '.mycommand <input>',
    ownerOnly: false,
    groupOnly: false,
    adminOnly: false,
    isPrefixless: true,
    cooldown: 5,

    async handler(sock, message, args, context = {}) {
        const {
            chatId, senderId, isGroup,
            isSenderAdmin, isBotAdmin,
            senderIsOwnerOrSudo, rawText,
            userMessage, config, channelInfo
        } = context;

        await sock.sendMessage(chatId, {
            text: `You said: ${args.join(' ')}`,
            ...channelInfo
        }, { quoted: message });
    }
};
```

🔧 Troubleshooting

Bot not connecting

· Verify SESSION_ID starts with Stanytz378/IAMLEGEND_
· If using PAIRING_NUMBER, link within 60 seconds of the code appearing
· Reset session and reconnect: npm run reset-session && npm start

myAppStateKey not present (pin/star broken)

Session lost its app state keys. Fix:

```bash
node -e "
const fs = require('fs');
const c = JSON.parse(fs.readFileSync('session/creds.json','utf8'));
delete c.myAppStateKeyId;
fs.writeFileSync('session/creds.json', JSON.stringify(c, null, 2));
console.log('Done');
"
npm start
```

Send any message to the bot – WhatsApp re-syncs keys automatically.

Commands not responding

· Check you're using the right prefix (default .)
· COMMAND_MODE=private → only owner can use commands
· OWNER_NUMBER must have no + sign

Data lost after restart

[!CAUTION]
Cloud platforms reset the filesystem on redeploy. Add MONGO_URL to use MongoDB – MongoDB Atlas has a free tier.

Port conflict

```bash
PORT=3000 npm start
```
